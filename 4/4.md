# 4.7 Módulo de Almacenamieto

### **Requerimientos del Módulo Inventario y materiales:**

| Secuencia | Actividad | Requerimientos del módulo |
| --- | --- | --- |
| 1 | Recepción de materiales | El sistema verifica automáticamente las cantidades y calidad en comparación con los pedidos originales, registrando discrepancias de forma automática. |
| 2 | Asignación digital de espacio de supervisión para el cliente | Configura áreas específicas para el control de calidad y permite la supervisión directa del cliente a través de permisos específicos. |
| 3 | Supervisión del cliente en producción | Facilita la supervisión del cliente mediante notificaciones automáticas y acceso a informes de producción. |
| 4 | Almacenamiento de materiales | Automatiza la asignación de espacios de almacenamiento según el tipo de material y las características requeridas. |
| 5 | Revisión de órdenes de producción | Genera automáticamente listas de materiales pendientes para la producción y revisa la urgencia de cada orden. |
| 6 | Evaluación automática y traslado de órdenes urgentes | El sistema evalúa la urgencia de las órdenes de producción y prioriza el traslado de materiales de acuerdo con estas. |
| 6.1 | Traslado de productos con órdenes más urgentes | Automatiza el envío de productos a las áreas de producción que requieren mayor urgencia. |
| 6.2 | Mantenimiento de productos en reserva | Monitorea de forma automática los productos en reserva y genera alertas cuando deben ser enviados a producción. |
| 7 | Revisión y aprobación del producto pintado | Automatiza el control de calidad de los productos pintados y facilita su aprobación o devolución para correcciones. |
| 8 | Traslado del producto pintado | Registra automáticamente el traslado del producto al área de expedición, controlando el estado de la entrega. |
| 9 | Decisión sobre la continuidad del proyecto | Facilita la toma de decisiones a través de un sistema de gestión de permisos para la supervisión del cliente en proyectos futuros. |
| 9.1 | Renovación de permisos de supervisión | Permite la actualización automática de los permisos de supervisión en base a las decisiones del cliente. |
| 9.2 | Denegación de permisos del proyecto | El sistema retira automáticamente los permisos y reasigna el espacio en almacén cuando un proyecto finaliza. |
| 10 | Control automatizado mensual de inventarios | Automatiza la revisión del inventario y genera pedidos de reabastecimiento en base a los niveles de stock actuales. |

### **Requisitos Funcionales del Módulo de Almacenamiento:**

1. **Recepción de Materiales**: Verificación automática de cantidades y calidad respecto a los pedidos originales, detectando discrepancias.
2. **Asignación de Espacio para Supervisión del Cliente**: Permite la supervisión directa del cliente mediante la asignación de áreas específicas en el almacén.
3. **Supervisión del Cliente en Producción**: Notificaciones automáticas y acceso a informes de producción para facilitar el seguimiento por parte del cliente.
4. **Almacenamiento de Materiales**: Asignación automatizada de espacios según el tipo de material y sus necesidades.
5. **Revisión de Órdenes de Producción**: Generación automática de listas de materiales pendientes y priorización de órdenes urgentes.
6. **Evaluación y Traslado de Órdenes Urgentes**: Priorización automática del traslado de materiales a producción según la urgencia.
7. **Mantenimiento de Productos en Reserva**: Monitoreo automatizado de productos en reserva con alertas para su envío a producción.
8. **Aprobación de Producto Pintado**: Control automático de calidad y gestión de devoluciones para correcciones si es necesario.
9. **Gestión de Entregas**: Registro y control automático del traslado de productos al área de expedición.
10. **Control de Inventario**: Revisión automática mensual del inventario y generación de órdenes de reabastecimiento cuando sea necesario.

### **Casos de uso:**

### Caso de uso N°1: Recepción de materiales

| **Objetivo** | Automatizar la recepción de materiales verificando cantidades y calidad automáticamente según los pedidos. |
| --- | --- |
| **Descripción** | El sistema recibe los materiales, los compara con los pedidos originales y genera alertas de discrepancias automáticamente. |
| **Actores Primarios** | Jefe de almacén, Almacenero. |
| **Precondiciones** | Sistema conectado al inventario en tiempo real, y pedidos previamente registrados. |
| **Paso** | **Acción** |
| 1 | El jefe de almacén registra la entrada de materiales. |
| 2 | El sistema verifica automáticamente las cantidades y la calidad. |
| 3 | Si hay discrepancias, se generan alertas y se notifica a los responsables. |
| 4 | Los materiales se registran en el sistema. |
| 5 | El caso finaliza cuando se validan todas las entradas. |

### Caso de uso N°2: Gestión de entregas

| **Objetivo** | Organizar y automatizar las órdenes de entrega según los pedidos. |
| --- | --- |
| **Descripción** | El sistema genera las órdenes de entrega y permite asignar cantidades y destinos automáticamente según las órdenes de clientes. |
| **Actores Primarios** | Jefe de almacén, Transportista. |
| **Precondiciones** | Pedidos aprobados y lista de productos disponibles en stock. |
| **Paso** | **Acción** |
| 1 | El sistema genera una orden de entrega. |
| 2 | El jefe de almacén valida la orden y asigna un transportista. |
| 3 | El sistema notifica al transportista sobre el envío y proporciona la información necesaria. |
| 4 | El transportista recibe los productos y confirma la entrega. |
| 5 | El sistema marca la orden como completada. |

### Caso de uso N°3: Ajuste de inventario

| **Objetivo** | Automatizar el ajuste de inventarios cuando haya diferencias entre el inventario físico y el registrado. |
| --- | --- |
| **Descripción** | Permite ajustar manual o automáticamente el inventario en base a las verificaciones físicas. |
| **Actores Primarios** | Jefe de almacén. |
| **Precondiciones** | Discrepancias entre el inventario físico y el sistema. |
| **Paso** | **Acción** |
| 1 | El jefe de almacén detecta una diferencia. |
| 2 | El sistema permite ajustar las cantidades de productos. |
| 3 | El sistema actualiza las bases de datos y genera un reporte del ajuste. |

### Caso de uso N°4: Reabastecimiento automatizado

| **Objetivo** | Generar órdenes de reabastecimiento automáticamente cuando se alcancen los niveles mínimos de inventario. |
| --- | --- |
| **Descripción** | El sistema monitorea los niveles de inventario y, al llegar al umbral mínimo, genera órdenes de compra o fabricación. |
| **Actores Primarios** | Sistema, Jefe de compras. |
| **Precondiciones** | Inventario en tiempo real y reglas de reabastecimiento configuradas. |
| **Paso** | **Acción** |
| 1 | El sistema detecta que un producto ha alcanzado el umbral mínimo. |
| 2 | El sistema genera automáticamente una orden de reabastecimiento. |
| 3 | El jefe de compras revisa y aprueba la orden. |
| 4 | El sistema envía la orden al proveedor o a producción. |
| 5 | El reabastecimiento es registrado en el inventario. |

### Caso de uso N°5: Informes de stock

| **Objetivo** | Generar informes detallados sobre el estado del inventario. |
| --- | --- |
| **Descripción** | El sistema proporciona informes automáticos sobre el stock real, disponible y pronosticado. |
| **Actores Primarios** | Jefe de almacén, Gerencia. |
| **Precondiciones** | Sistema de inventario conectado a los informes. |
| **Paso** | **Acción** |
| 1 | El usuario solicita un informe de inventario. |
| 2 | El sistema genera un reporte con los niveles de stock real, disponible y pronosticado. |
| 3 | El usuario puede exportar o visualizar el informe. |
