# 4.1. Módulo Gestion de Poyecto
## Requerimientos del Módulo Gestión de Proyectos:

| Secuencia | Actividad | Requerimientos del módulo |
| --- | --- | --- |
| 1 | Crear proyecto | El sistema debe permitir crear nuevos proyectos, asignando un código único, definiendo metas específicas y fechas de inicio y fin. |
| 2 | Asignar equipo a proyecto | El sistema permite asignar personal de producción, logística y calidad, tomando en cuenta la carga de trabajo actual y la disponibilidad de cada recurso. |
| 3 | Definir hitos del proyecto | El sistema genera los hitos clave del proyecto basados en los tiempos de ejecución y fabricación de Castro Contratistas, permitiendo ajuste manual por parte del gerente de proyectos. |
| 4 | Monitorear progreso en tiempo real | El sistema permite al gerente de proyectos y al jefe de producción monitorear el estado de las tareas en tiempo real, con alertas si alguna actividad se retrasa o se adelanta. |
| 5 | Verificar cumplimiento de hitos | El sistema verifica si los hitos se cumplen en el plazo establecido y envía notificaciones automáticas a los responsables en caso de retrasos. |
| 6 | Evaluar riesgos del proyecto | El sistema permite al gerente de proyectos identificar riesgos (retrasos en materiales, sobrecarga de trabajo) y generar un plan de mitigación. |
| 7 | Reasignar recursos por riesgos | Basado en los riesgos identificados, el sistema sugiere reasignaciones de personal y recursos para garantizar el cumplimiento de los hitos críticos. |
| 8 | Revisar entregables | El sistema facilita la revisión de los entregables por el cliente y personal interno, con la posibilidad de aprobar o solicitar correcciones. |
| 9 | Cerrar proyecto | El sistema gestiona el cierre del proyecto, generando reportes finales y almacenando toda la documentación en el archivo digital de la empresa. |
| 10 | Generar informes de desempeño | El sistema genera informes automáticos que incluyen tiempos de entrega, cumplimiento de hitos, costos y recursos utilizados, todo integrado al sistema de control de calidad de Castro Contratistas. |

## Requisitos funcionales:

### Creación de proyectos:
- El sistema debe permitir la creación de nuevos proyectos específicos de ingeniería, asignando un código único y estableciendo objetivos claros.
- Los proyectos deben estar alineados con los estándares de Castro Contratistas en cuanto a tiempos de producción y materiales.

### Asignación de recursos:
- Asignación automática de equipos de trabajo basados en la disponibilidad de personal en áreas clave como producción, logística y control de calidad.
- El sistema sugiere asignaciones automáticas en función de la carga actual de trabajo y la disponibilidad de recursos, aunque el gerente de proyectos puede hacer ajustes manuales.

### Monitoreo del progreso:
- El sistema permite el monitoreo en tiempo real del estado de las tareas y el progreso en el taller de producción.
- Debe enviar alertas automáticas al gerente de proyectos y al jefe de producción en caso de retrasos o problemas en la fabricación.

### Gestión de riesgos:
- El sistema permite la identificación de riesgos en el proceso, tales como retrasos en la entrega de materiales o fallos en las máquinas de producción.
- El gerente de proyectos puede definir planes de contingencia para mitigar estos riesgos y asignar recursos adicionales si es necesario.

### Aprobación de entregables:
- El sistema notifica a los responsables internos y al cliente cuando los entregables están listos para su revisión.
- Debe permitir la aprobación de entregables o la solicitud de ajustes y correcciones, generando un historial de aprobaciones.

### Informes automáticos:
- El sistema genera informes detallados al finalizar el proyecto, que incluyen el uso de recursos, el tiempo invertido en cada tarea, el cumplimiento de hitos y la calidad de los resultados.

## Requisitos no funcionales:

### Interfaz amigable y adaptada a la empresa:
- La interfaz debe ser fácil de usar y personalizada para los procesos de Castro Contratistas, permitiendo una navegación eficiente entre las tareas y proyectos.

### Seguridad:
- Solo usuarios autorizados, como el gerente de proyectos, jefe de producción y personal de calidad, deben tener acceso a la creación y modificación de proyectos.
- Se deben implementar diferentes niveles de permisos según el cargo.

### Rendimiento:
- El sistema debe procesar la asignación de recursos, actualización de tareas y generación de informes sin retrasos, incluso cuando se manejen varios proyectos grandes al mismo tiempo.

### Notificaciones automáticas:
- Deben ser claras y enviadas a tiempo, tanto a los gerentes de proyectos como a los responsables de áreas clave, avisando sobre retrasos o problemas en la producción.

### Acceso desde cualquier dispositivo:
- El sistema debe ser accesible desde dispositivos móviles y computadoras para permitir el seguimiento y control remoto del avance de los proyectos.

## Usuarios del Módulo:

- **Gerente de Proyectos**: Responsable de la creación, seguimiento y control del avance del proyecto.
- **Jefe de Producción**: Responsable de asignar y controlar las tareas en el taller, monitoreando los avances y el uso de recursos.
- **Equipo de Trabajo**: Personal de producción, calidad y logística encargado de ejecutar las tareas asignadas y reportar avances.
- **Cliente**: En algunos casos, puede acceder al sistema para revisar el estado de los entregables y aprobarlos.

## Información gestionada por el Módulo:

- **Datos del proyecto**: Código del proyecto, nombre, fechas de inicio y fin, descripción de metas y objetivos.
- **Hitos y entregables**: Fechas programadas y reales, estado de los entregables, aprobación o rechazo.
- **Recursos humanos**: Disponibilidad del personal, asignación de tareas, carga de trabajo, cambios en las asignaciones.
- **Riesgos del proyecto**: Evaluación de riesgos (como retrasos en materiales o fallos técnicos), planes de mitigación y reasignación de recursos.
- **Reportes**: Informes sobre el desempeño del proyecto, cumplimiento de hitos, costos y tiempos.

## Casos de uso:

### Caso de uso N°1: Crear proyecto

| **Objetivo** | Automatizar la creación de proyectos específicos de ingeniería, asignando recursos iniciales y plazos de entrega. |
| --- | --- |
| **Descripción** | El sistema permite al gerente de proyectos crear un nuevo proyecto con todos los detalles específicos de Castro Contratistas, asignando recursos y plazos de entrega iniciales. |
| **Actores Primarios** | Gerente de Proyectos, Jefe de Producción. |
| **Precondiciones** | Recursos disponibles y plantillas predefinidas para los tipos de proyecto de la empresa. |
| **Paso** | **Acción** |
| 1 | El gerente de proyectos solicita la creación de un nuevo proyecto en el sistema. |
| 2 | El sistema asigna un código único y asigna recursos humanos y materiales. |
| 3 | Se define el plan inicial basado en las especificaciones del cliente. |
| 4 | Se genera el proyecto y el caso finaliza cuando se asignan las primeras tareas. |

### Caso de uso N°2: Monitorear progreso del proyecto

| **Objetivo** | Permitir el monitoreo en tiempo real del avance de las tareas y alertar sobre problemas o retrasos. |
| --- | --- |
| **Descripción** | El sistema actualiza el progreso de las tareas en tiempo real y envía alertas cuando se detectan problemas o retrasos en la producción. |
| **Actores Primarios** | Gerente de Proyectos, Jefe de Producción. |
| **Precondiciones** | Proyecto y tareas previamente creadas y asignadas a los equipos de trabajo. |
| **Paso** | **Acción** |
| 1 | El sistema actualiza el estado de las tareas en tiempo real. |
| 2 | Envía alertas en caso de retrasos o problemas detectados. |
| 3 | El gerente de proyectos puede reasignar recursos si es necesario para cumplir con los plazos. |
