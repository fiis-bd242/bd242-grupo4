# Módulo de Gestión de Proyectos

### Lista de Proyectos

El jefe de proyectos puede visualizar una lista de todos los proyectos existentes, con opciones para actualizarlos o eliminarlos según sea necesario.

![Imagen1](https://github.com/user-attachments/assets/577a5803-4e5e-4903-8abc-a684cd7adeb8)


### Crear Proyecto

El jefe de proyectos inicia el proceso de creación de un nuevo proyecto, proporcionando detalles como el nombre, fecha de inicio, fecha de fin y estado.

![Imagen2](https://github.com/user-attachments/assets/3961a0c8-90e1-444c-876c-ae154659ecab)

### Rellenar Detalles del Proyecto

El jefe de proyectos rellena todos los campos necesarios, como el ID del proyecto, nombre, fechas de inicio y fin, y estado.

![Imagen3](https://github.com/user-attachments/assets/ef7a53ea-21b5-4647-a5be-93d64e6c0c3e)


### Confirmación de Eliminación de Proyecto

Cuando el jefe de proyectos decide eliminar un proyecto, se muestra un cuadro de confirmación para evitar acciones accidentales.

![Imagen4](https://github.com/user-attachments/assets/e0ab4d59-a983-4f4e-bc4a-64f5ae8f4443)

# módulo de Mantenimiento

### Módulo de Gestión de Equipos
1. **Pantalla de inicio de sesión**: Los usuarios autenticados pueden acceder al sistema.
![inicio](image-5.png)
2. **Pantalla de listado de equipos**: Vista de todos los equipos registrados, con filtros por tipo, estado y ubicación.
![equipos](image-6.png)
3. **Pantalla de detalle de equipo**: Detalles completos del equipo seleccionado, con la opción de agregar un nuevo equipo o editar el existente.
![detalle-equipo](image-7.png)

### Módulo de Gestión de Infraestructura
1. **Pantalla de listado de infraestructura**: Muestra todas las infraestructuras disponibles, con filtros para la ubicación y estado.
![lista-infraestructura](image-9.png)
2. **Pantalla de detalle de infraestructura**: Información detallada de la infraestructura seleccionada, y la opción de agregar o editar detalles.
![detalle-infraestructura](image-10.png)

### Módulo de Solicitudes de Mantenimiento
1. **Pantalla de solicitudes de mantenimiento**: Permite al operario generar una solicitud para el mantenimiento de un equipo o infraestructura, con campos para la descripción, urgencia y equipo asociado.
![solicitar-mantenimiento](image-8.png)
2. **Pantalla de cronograma de mantenimiento**: Visualización de los cronogramas de mantenimiento programados con opción para editar fechas y asignar recursos.
![cronogramas](image-11.png)

# Modulo Inspeccion

Empezaria el modulo con el panel de control que se ve del modulo:

![image](https://github.com/user-attachments/assets/f54471fd-f24d-4b00-9680-b0c9671c0452)

Empezariamos entrando a Verificacion documental, viendo que el sistema ya tiene registrado los materiales:

![image](https://github.com/user-attachments/assets/e516b780-4593-450d-afef-a64a7652657e)

Luego, entraria a un material para hacerle dicha verificación, rellenando información para cumplir las normas preestablecidas

![image](https://github.com/user-attachments/assets/664b9ed1-bd22-4ed1-becb-b3a70ab0a1f4)

Dandole a siguiente, se decidiría si es aprobado o no la verificación de acuerdo a si cumplió con las normas, dando una justificación y, dandole a siguiente denuevo, ya estaria registrado la verificación en el sistema:

![image](https://github.com/user-attachments/assets/29af1190-037d-41b5-9c98-9189aeca1693)
![image](https://github.com/user-attachments/assets/922e77b1-d9d1-4f68-8549-5314fb5d6be8)

Luego, se podria entrar también a la Inspección, sobre todo dimensional, entrando a la interfaz y viendo el sistema de materiales:

![image](https://github.com/user-attachments/assets/3a11817e-169d-4870-89b2-0172d3d723e8)

Después, tendremos que escoger el material a inspeccionar y tendremos que escoger si queremos un control dimensional, o control de composición, el cual al entrar deberemos ingresar informacion de esos formularios, teniendo a la vista la ficha tecnica del material:

![image](https://github.com/user-attachments/assets/8d83a8bc-96df-4c9e-bd58-1347a880e08d)

![image](https://github.com/user-attachments/assets/7f9ef4b6-e345-4644-a6c5-dad994da49ef)

Luego, en ambas se decidira si se aprueba o rechaza las especificaciones tecnicas, dando una justificación si se rechaza para saber el motivo, registrandose en el sistema y actualizandose su estado:

![image](https://github.com/user-attachments/assets/58f59a37-8dd3-405e-a392-63f8f0191042)

![image](https://github.com/user-attachments/assets/33d5f9b1-b99d-4f6f-8f5b-061ff59448f2)

Luego, se abrira un panel para ver si se aprueba o no el resultado de estas dos actividades anteriores, siendo necesariamente que ambas sean aprobadas para dar como resultado un aprobado.
